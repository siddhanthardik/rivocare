# RIVO - Home Healthcare Platform

A production-grade MERN stack application connecting patients with home healthcare professionals (nurses, doctors, physiotherapists, caretakers).

## Features
- **Booking Machine**: State machine for booking workflow (Pending -> Confirmed -> In Progress -> Completed/Cancelled).
- **Three-Tier RBAC**: Distinct Dashboards for Patients, Providers, and Admins.
- **Provider Analytics**: Dashboard tracking earnings, pending requests, and upcoming bookings.
- **Provider Verification**: Admin verification workflow for healthcare professionals.

## Tech Stack
- **Frontend**: React (Vite), Tailwind CSS, React Router DOM v6, React Hot Toast, Axios
- **Backend**: Node.js, Express, MongoDB/Mongoose, JWT, bcryptjs, morgan

## Prerequisites
- Node.js 18+
- MongoDB instance (local or Atlas)

## Quick Start

### 1. Install Dependencies
Install dependencies concurrently for both backend and frontend:
```bash
npm install
```

### 2. Environment Setup
The agents have automatically generated `.env` files for both `backend` and `frontend`. 
Ensure your MongoDB is running locally at `mongodb://localhost:27017/rivo` or update `backend/.env`.

### 3. Database Seeding
To populate the database with services, an admin account, a demo patient, and a demo provider:
```bash
npm run seed
```

### 4. Run Application
Run both backend and frontend concurrently:
```bash
npm run dev
```

## API Endpoints (Core)
- `POST /api/auth/register` - Register a new user
- `POST /api/auth/login` - Authenticate
- `GET /api/bookings` - Get user's/provider's bookings
- `POST /api/bookings` - Create a booking
- `PUT /api/bookings/:id/status` - Update booking status
- `GET /api/providers` - Find providers (filtered by pincode + service)
- `PUT /api/providers/availability` - Toggle provider online/offline status
